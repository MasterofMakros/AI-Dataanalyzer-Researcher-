# Generate PRP

**Step 1: Read INITIAL.md**
Read the `INITIAL.md` file in the root directory to understand the request.

**Step 2: Create PRP File**
Create `PRPs/<feature-slug>.md` using the template `PRPs/templates/prp_base.md`.

**Step 3: Fill Details**
- Map `FEATURE` -> `Goal` & `Requirements`.
- Map `EXAMPLES` -> `Context / Repo Patterns`.
- Develop the **Implementation Plan** based on the architecture.

**Step 4: Review**
Present the PRP to the user for approval.
